# Auto-Work Risk Gates / 自动工作风险闸门

本文件定义 `auto-work` 在执行型任务中的高风险动作边界。

## 1. 风险闸门的目标

risk gates 的目标是：
1. 防止执行型任务越权推进
2. 防止在高风险节点“默认继续”
3. 让自动工作只在安全边界内持续推进
4. 在需要人工拍板时及时停下

## 2. 必须停下确认的高风险动作

以下动作一律视为硬闸门，必须等待明确确认：

### 2.1 deploy
- 发布到生产 / 线上
- 推送正式环境变更
- 更新线上服务配置

### 2.2 restart
- 重启服务
- 重启容器
- 重启守护进程
- 重启网关 / agent / node / system service

### 2.3 delete
- 删除文件
- 删除目录
- 删除数据
- 删除配置项
- 覆盖性清空或不可逆清理

### 2.4 auth-route / 鉴权路由变更
- 修改鉴权中间件
- 放开 / 收紧认证
- 改变白名单 / 黑名单逻辑
- 修改路由访问边界
- 改变登录、权限、令牌校验路径

## 3. 默认可继续的低风险动作

以下动作通常可继续自动推进：
- 读取文件
- 阅读日志
- 搜索代码
- 修改本地代码
- 修改未生效配置文件
- 写测试
- 跑测试
- 跑 lint
- 本地 build
- 本地验证
- 生成草稿、补丁、diff、方案文档

## 4. 风险闸门的触发时机

一旦判断“下一步动作将进入高风险节点”，必须立即停止自动续跑，并切换到等待确认。

硬规则：
- **风险闸门必须前置，不得后置。**

## 5. 进入风险闸门后的正确行为

1. 停止进一步自动推进
2. 明确说明即将做什么
3. 明确说明为什么这是高风险动作
4. 说明当前已完成到哪一步
5. 等待用户明确确认

## 6. 判断不清时的默认策略

当某个动作是否属于高风险不够确定时：
- **宁可停下确认，也不要默认继续。**
