# MEMORY.md - Long-Term Memory

## Preferences / Operating rules

- 默认工作入口规则（2026-03-12）：**凡是“不是一句话就能收尾”的任务，默认进入 `auto-work`。**
- 默认命中范围（满足任一即可）：
  - 预计超过 5 分钟的任务
  - 多步骤 / 多文件 / 多轮推进任务
  - 代码修改、修 bug、重构、新功能、脚本/配置/部署类任务
  - 运维 / 排障 / 远程机处理 / 需要反复验证状态的任务
  - 用户明确表达“持续做完 / 做完再停 / 不要只汇报 / 不要只监督，直接做 / 继续推进 / 自动工作”
  - 用户明确表达以下强触发短语时，必须直接视为命中 `auto-work`：
    - `继续，采用自动工作`
    - `继续采用自动工作`
    - `继续，用自动工作`
    - `继续，按自动工作做`
    - `继续推进，进入自动工作`
    - `继续，进入自动工作`
  - 容易挂机、空转、断链、需要持续接力的任务
- 默认不自动进入 `auto-work` 的范围：
  - 单轮问答、简单解释、翻译、一次性建议
  - 纯讨论 / 先分析方案、不立即执行
  - 高风险待拍板动作（deploy / restart / reload / 删除 / 鉴权 / 路由 / 防火墙 / 权限边界改动）
- 执行原则：
  - 命中上述范围时，主流程默认按 `auto-work` 处理，不必每次单独征求是否开启
  - 但高风险动作即使处于 `auto-work` 中，仍必须单独确认
  - 对外话术统一用 `auto-work`，不再用旧 `code-moment` / `task-pulse` 心智
